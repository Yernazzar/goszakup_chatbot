Архитектура Системы Госзакупок

Интеллектуальный анализ аномалий и справедливая оценка цен

1. Модульная Схема

graph TD User((Пользователь)) <--> Web[Flask Web App] Web <--> Agent[AI Agent - LangChain] subgraph "Logic & Tools" Agent --> Tools[Custom Tools] Tools --> FP[Fair Price Engine] Tools --> Search[Semantic Vector Search] end subgraph "Data Layer" Search <--> DB[(PostgreSQL + pgvector)] FP <--> DB DB --- Marts[Analytical Marts] end subgraph "Batch Processing" Airflow[Apache Airflow] --> ETL[ETL Pipelines] ETL --> DB ETL --> Classify[ENSTRU Classifier] end

2. Технологический Стек

Интерфейс Flask, HTML5, CSS3, JS (Modern UI)
Искусственный Интеллект LangChain, OpenAI (GPT-4o), ReAct Pattern
База Данных PostgreSQL 16 + pgvector (Векторный поиск)
Оркестрация Apache Airflow (ETL & ML Pipelines)

3. Аналитические Метрики

Основная метрика — Fair Price (Справедливая цена), рассчитываемая по формуле:

Fair Price = Baseline × Region × Inflation × Seasonality
  • Baseline: Медианная цена категории ЕНСТРУ.
  • Region: Коэффициент региона (КАТО).
  • Inflation: Поправка на дату заключения контракта.
  • Seasonality: Учет сезонных колебаний спроса.

4. Риски и Ограничения

  • Качество данных: Ошибки в наименованиях в первоисточнике.
  • LLM Hallucinations: Минимизированы через строгий SQL Execution.
  • Задержка: Аналитика базируется на подписанных контрактах (лаг 1-2 недели).
